The Architecture

How It Works

network_ping High-Speed Sharding

For O(1) complexity, the 14.3M record database is split into 16 compressed Gzip shards based on the first character of the hash. We further index these using a **4-character hex prefix**.

1. Calculate Prefix (e.g. 6d00)
2. HTTP Byte-Range Request (~30KB snippet)
3. Instant Match result

This makes lookups instant even on limited bandwidth by avoiding full file downloads.

psychology Parallel Scanning Cluster

Since indexing SHA1/SHA256 is too bulky for static hosting, we implement a background scanning cluster that uses parallel hardware acceleration.

  • • Spawns 4 WebWorkers as background threads.
  • • Streams highly-compressed wordlist in 4 segments.
  • • Scans on-the-fly, keeping the UI at 60fps.
